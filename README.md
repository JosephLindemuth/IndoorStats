# Indoor League Stats

An unofficial stats dashboard for the Bluegrass Ultimate indoor leagues. Data is scraped from [bluegrassultimate.org](https://bluegrassultimate.org) and displayed as a web app with team and player statistics.

Pre-scraped data viewable here: https://josephlindemuth.github.io/IndoorStats/

---

## What It Does

- Scrapes team rosters and game results from each indoor league season on bluegrassultimate.org
- Generates two CSV files: `league_rosters.csv` and `league_results.csv`
- A web dashboard (`index.html`) reads those CSVs and displays:
  - A sortable table of all players with career stats
  - A sortable table of all teams with season stats
  - Individual team pages with roster and game log
  - Individual player pages with career stats and a "compare with" tool to see head-to-head and played-together stats against any other player

## Scraping Data

### Requirements

```
pip install requests beautifulsoup4 python-dotenv
```

### Credentials

Create a `.env` file in the same directory as the scraper with your bluegrassultimate.org login:

```
LEAGUE_EMAIL=your@email.com
LEAGUE_PASSWORD=yourpassword
```

### Running the Scraper

```
python main.py
```

This will log in, scrape all configured league seasons, and write two files to the current directory:

- `league_rosters.csv` — one row per player per team, with league, team name, player name, and roles
- `league_results.csv` — one row per game, with league, date, home/away teams, scores, and results

The scraper is configured to pull from these six seasons:

| Season | URL slug |
|--------|----------|
| 2024 — Indoor Session 1 | `2024-indoor-league` |
| 2024 — Indoor Session 2 | `indoor-session-2-the-eras-league` |
| 2025 — Indoor Session 1 | `indoor-league-2025-session-1` |
| 2025 — Indoor Session 2 | `indoor-league-2` |
| 2026 — Indoor Session 1 | `2026-indoor-session-1` |
| 2026 — Indoor Session 2 | `2026-indoor-session-2` |

To add or remove seasons, edit the `LEAGUE_URLS` list at the top of `main.py`.

## Viewing the Dashboard

### Why you can't just open index.html

Browsers block web pages from reading local files directly due to CORS (Cross-Origin Resource Sharing) security restrictions. If you open `index.html` by double-clicking it, it will fail to load the CSV files.

The fix is to serve the files through a simple local web server.

### Starting a local server

Make sure `index.html`, `style.css`, `app.js`, `league_rosters.csv`, and `league_results.csv` are all in the same folder. Then open a terminal in that folder and run:

**Python 3**
```
python -m http.server 8080
```

**Node.js (alternative):**
```
npx serve .
```

### Opening the app

Once the server is running, open your browser and go to:

```
http://localhost:8080
```
